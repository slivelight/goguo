## goguo产品
1. 电梯演讲
“你有没有遇到过：想看的网站，在家里能打开，一到公司或酒店就不行了？
那是被墙挡了。
过得去 —— 一个小工具，专门帮你在不同网络环境里，自动让那些被墙的网站‘过得去’。装上之后，你换地方，它自动切换策略，被墙的网站继续能用。如果万一没自动立即修好,也会提醒你，同时有后台监控程序，尽快帮你自动修复，用户无感。
被墙的，过得去；本来能上的，更顺畅。上网，不该被墙打断。”
2. 产品宣传文稿
产品名称:过得去
英文名：GoGuo（音译，保留原味，也可意译为 PassWall，但拼音更独特）
产品理念: “被墙的，过得去；上网，就该这么自然。”
专治被墙：核心能力就是让各种网络环境下（家、公司、酒店、机场）的被墙网站自动保持可访问。
自动跟随：你切换Wi-Fi，它自动切换代理方案，无需任何设置。
3. 典型用户场景:
场景A（家→公司）：早上在家能看某国外新闻网站，到公司连上Wi-Fi后，该网站打不开。“过得去”自动识别网络变化，重新建立通道，1秒后恢复正常。用户无感。
场景B（出差酒店）：入住后发现常用的学术网站（被墙）无法访问。“过得去”弹窗提示“自动修复中”，显示一个修复进度条，修复完后，问题解决。
场景C（办公室开发者 — 每日网络切换）：
```
08:00 到公司 → 连接企业 WiFi (10.x.x.x)
  → 系统检测到企业网络 → 自动切换到代理模式 (VLESS+Reality 穿透企业防火墙)
  → GitHub 可用 ✓

12:00 午休 → 切到手机热点 (100.64.x.x)
  → 系统检测到移动网络 → 自动切换到 Hysteria2 (QUIC 抗抖动)
  → GitHub 可用 ✓ (无感切换，用户不知情)

18:00 回家 → 连接家庭 WiFi (192.168.x.x)
  → 系统检测到家庭网络 → 自动切换到 Hosts 模式 (最省资源)
  → GitHub 可用 ✓

22:00 咖啡厅 → 连接公共 WiFi (172.16.x.x)
  → 系统检测到未知/公共网络 → 强制代理模式 (加密防窃听)
  → GitHub 可用 ✓
```
场景D：移动办公 — 不稳定网络
```
高铁上 → 手机热点信号不稳定
  → 当前节点 JP-Hy2 超时 → 自动切换到 KR-Hy2 (同协议不同节点)
  → KR-Hy2 也超时 → 自动切换到 SG-VMess (不同协议)
  → SG-VMess 可用 ✓
  → 全程用户无感知

进入隧道 → 完全断网
  → 检测到 claude 不可达 → 暂停监控 (避免无效重试消耗资源)
  → 网络恢复 → 自动重新探测 → 恢复最优方案
```
场景E：自动恢复失败 — 手动介入
```
某天代理节点全部失效 → 自动恢复尝试全部失败
  → 窗口弹出醒目提示:
    "⚠ google 访问中断 — 当前方案: 代理 (所有节点不可达)"
    "自动恢复已尝试 3 次，建议手动处理:"
    "  1. 更新订阅节点"
    "  2. 切换到 Hosts 模式"
    "  3. 运行网络诊断"
    "  4. 查看详细日志"
  → 用户按 1 更新订阅 → 新节点获取 → 自动恢复 ✓
```
