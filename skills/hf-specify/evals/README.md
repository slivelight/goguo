# 需求阶段评测

这个目录包含 `hf-specify` 的评测 prompts。

## 目的

这些评测用于验证需求阶段是否真正做到：

- 先澄清再起草规格
- 不提前进入设计
- 正确把草稿交给 `hf-spec-review`
- 在 review 回修阶段，只把真正需要用户裁决的问题带回给用户

## Fixture-Backed Cases

部分评测会在 `files` 字段里附带真实 fixture 文件，用来模拟：

- 当前轮范围与待确认边界
- 规格草稿与 review 记录
- 回修时哪些问题可直接修，哪些问题必须先确认

## 建议评分关注点

1. 是否主动澄清范围、约束、验收标准
2. 是否避免提前输出设计方案
3. 是否把结果交给 `hf-spec-review`
