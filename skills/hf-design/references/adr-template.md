# ADR 模板

对每个会影响后续任务规划的关键决策，使用以下格式记录。

决策记录的详细程度应与决策的影响力匹配：影响全局架构的决策写完整 ADR，局部技术选型可以简化为一段决策说明。

## ADR 格式

```markdown
## ADR-{编号}: {决策标题}

### 状态
[提议 | 已接受 | 已废弃 | 被替代]

### 背景
[为什么需要这个决策？当前约束和驱动力是什么？]

### 决策
[清楚描述选择了什么]

### 被考虑的备选方案
[列出被否决的方案及否决理由]

### 后果
- 正面：[收益]
- 负面：[代价]
- 中性：[既不好也不坏的附带影响]

### 可逆性评估
[容易回滚 / 中等成本 / 高成本重构]
```

## 使用建议

- 高可逆性决策可以快速推进，不需要完整 ADR
- 低可逆性决策必须在设计阶段充分论证，写完整 ADR
- ADR 编号按设计文档内出现顺序递增
