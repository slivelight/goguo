# 可追溯性评审评测

这个目录包含 `hf-traceability-review` 的评测 prompts。

## 目的

这些评测用于验证追溯评审是否真正做到：

- 检查规格→设计→任务→实现→验证的完整证据链
- 发现断链或漂移时给出明确 verdict
- 不以"代码能跑"等同于"追溯完整"
- 在 route / stage / evidence 冲突时先做 precheck 阻塞
- 显式指出需要回写或同步的工件

## 建议评分关注点

1. 是否逐层检查追溯链而非只看代码
2. 是否发现断链后给出正确 reroute 或回修方向
3. 是否给出唯一 canonical 下一步
4. 是否指出回写义务，而不只说“有漂移”
