# Multi-Agent 头脑风暴辩论协议

用于 hf-strategy-discovery 的辩论流程详细说明。

---

## 一、协议概述

核心思想：
- 通过正方和反方的辩论，暴露战略方案的潜在风险和机会
- 通过评委综合，量化论据强度和决策置信度
- 最终收敛形成更稳健的战略方案

---

## 二、激活规则

| Profile | 激活条件 | 激活时机 |
|---------|---------|---------|
| **lightweight** | 用户主动要求 | Phase 2 用户交互时明确要求 |
| **standard** | 用户提出疑虑或有争议决策 | Phase 2 用户反馈中包含疑虑 |
| **full** | 所有争议决策强制激活 | Phase 3 三定决策时强制触发 |

争议决策识别信号：
- 用户说"我不确定这个方向是否可行"
- 用户说"有没有考虑风险？"
- 多个机会方向难以用SPAN矩阵唯一排序

---

## 三、辩论流程

### Phase 1：初步方案生成

**执行者**：主 Agent

**输出**：战略方案 v1

```markdown
## 战略方案 v1

### 核心战略方向
[一句话描述]

### 关键假设
| 假设ID | 假设内容 | 来源 | Confidence |
|--------|----------|------|------------|
| HYP-1 | [假设] | [来源] | 高/中/低 |
```

### Phase 2：用户交互反馈

**主 Agent 动作**：
1. 向用户展示战略方案 v1
2. 收集用户反馈：
   - "你认为这个方案有哪些疑虑？"
   - "有没有考虑到的风险？"

**触发判断**：用户反馈包含疑虑 → 激活辩论

### Phase 3：正方论证

**执行者**：proponent Agent（正方）

**论证框架**：

| 论点 | 内容 |
|------|------|
| 市场机会论证 | TAM/SAM/SOM + 增长趋势 + 时机窗口 |
| 能力匹配论证 | 技术能力 + 运营能力 + 资源匹配 |
| 执行路径论证 | 里程碑 + 依赖关系 + 风险缓解 |

**输出格式**：

```markdown
## 正方论证报告

| 论点 | 评分(1-10) | 说明 |
|------|-----------|------|
| 市场机会 | X | - |
| 能力匹配 | X | - |
| 执行路径 | X | - |
| **综合** | X | - |

**建议**: 采纳/有条件采纳
**置信度**: X%
```

### Phase 4：反方质疑

**执行者**：challenger Agent（反方）

**质疑框架**：

| 质疑 | 内容 |
|------|------|
| 市场风险 | 假设是否高估？客户是否愿意买单？ |
| 执行风险 | 技术是否成熟？资源是否低估？ |
| 竞争风险 | 竞争对手如何反击？护城河是否够深？ |

**输出格式**：

```markdown
## 反方质疑报告

| 风险类别 | 高风险项 | 致命风险 |
|----------|----------|----------|
| 市场风险 | [项] | [致命项] |
| 执行风险 | [项] | [致命项] |
| 竞争风险 | [项] | [致命项] |

**建议**: 采纳/修改后采纳/否决
**置信度**: X%
```

### Phase 5：评委综合

**执行者**：arbiter Agent（评委）

**输出格式**：

```markdown
## 评委综合评判报告

| 角度 | 正方强度 | 反方强度 | 优势方 |
|------|----------|----------|--------|
| 市场分析 | X/10 | X/10 | - |
| 能力匹配 | X/10 | X/10 | - |
| 执行可行性 | X/10 | X/10 | - |

**最终建议**: 采纳/有条件采纳/拒绝
**置信度**: X%
```

### Phase 6：方案修订

**执行者**：主 Agent

**输出**：战略方案 v2

## 四、辩论质量规则

### 论据质量标准

| 评分 | 标准 |
|------|------|
| 9-10 | 有充分数据支撑 + 成功案例可比 |
| 7-8 | 有较好数据支撑 + 案例 |
| 5-6 | 有一定数据支撑 |
| 3-4 | 数据不足，依赖假设 |
| 1-2 | 缺乏证据，纯直觉判断 |

### 置信度区间含义

| 置信度 | 含义 | 建议 |
|--------|------|------|
| ≥80% | 高置信 | 可以采纳 |
| 60-80% | 中置信 | 有条件采纳，需验证假设 |
| <60% | 低置信 | 需要更多验证 |

---

## 五、Profile 差异

| Profile | 辩论阶段 | 论据密度 | 输出长度 |
|---------|----------|----------|----------|
| lightweight | Phase 3-5（简化） | 1论点 + 1质疑 | ≤500字 |
| standard | Phase 1-6（完整） | 3论点 + 3质疑 | 1000-3000字 |
| full | Phase 1-6（完整） | 3论点 + 3质疑 + 博弈模拟 | 3000+字 |
