## 1. 设计系统

### 1.1 设计原则

1. **默认友好，进阶可达**：普通用户看到摘要即可放心（"所有目标站点可用"），进阶用户可展开查看细节（节点、规则、日志）
<!-- [?] id:01;status:close;date:2026-05-15T10:30  "信息密度优先"与战略发现中的"普通办公用户85%"画像存在潜在张力。建议补充一条原则："默认友好，进阶可达"——普通用户看到摘要即可放心，进阶用户可展开查看细节。当前原则列表缺少对"用户认知负担"的明确约束；回答：采纳。原"信息密度优先"重述为"默认友好，进阶可达"，兼顾信息密度与认知负担。-->
2. **渐进披露**：默认展示摘要，进阶操作按需展开。披露层次定义：
   - **第一层（仪表盘摘要）**：服务状态、站点可达性一句话结论、最近通知
   - **第二层（分类详情页）**：站点列表、规则列表、节点池表格
   - **第三层（单项详情弹窗/面板）**：站点诊断详情、规则域名展开、节点元数据
<!-- [TODO] id:02;status:close;date:2026-05-15T10:30  渐进披露的具体层次定义缺失。建议补充：第一层（仪表盘摘要）→第二层（分类详情页）→第三层（单项详情弹窗/面板）。这与战略发现 JTBD 中"进阶用户查看当前站点、节点、模式和日志"的需求对应。；任务处理结果：已补充三层披露定义。-->
3. **操作安全**：以下四类敏感操作需二次确认：
   - 服务启停（Feature 001 FR-2.2.2-R3）
   - Baseline 确认（Feature 001 FR-2.2.2-R3）
   - 规则应用（Feature 003 FR-2.6.1-R1）
   - 部署模式切换（Feature 002，影响活跃适配器列表）

4. **一致性**：Windows 和 WSL/Linux 两侧操作界面完全一致
5. **恢复信心可见**：用户随时能看到恢复状态、恢复进度、恢复失败时的下一步建议（对应战略发现核心 wedge："可停用、可恢复、可解释"）

6. **意图优先，技术隐藏**：用户只需指定"要访问哪些网站"，规则类型、代理协议、节点等技术细节默认隐藏，只在进阶模式下可见

- **默认尺寸**：960 × 640
- **最小尺寸**：800 × 480
- **标题栏**：自定义（Tauri 无边框 + 拖拽区域）。保留自定义标题栏——减少系统原生标题栏的视觉干扰，Tauri 拖拽区域对用户无额外认知负担

**添加站点交互**：

1. 输入站点标识（如 `github`）→ 匹配站点定义 → 展开域名预览
2. 输入域名（如 `example.com`）→ FR-2.1.1-R6 自动检测相关域名 → 展示建议列表
3. 用户确认 → 站点添加完成 → 自动触发规则重新生成

**预设模板交互**：

1. 选择模板 → 展示包含的站点列表
2. 用户可在模板基础上增删
3. 确认后批量添加
